Component Usage Notes

Overview

The Content/2ColumnGraphic component is a versatile page section designed to drive engagement and conversions. It features a large background image with either one or two call-to-action cards overlaid.

Variants

  • Orientation:
    • "left" - Single card aligned to the left
    • "twocolumns" - Two cards side by side (stacks on mobile)
  • Breakpoints:
    • Mobile: 380-767px (cards stack vertically)
    • Tablet: 768-1149px (reduced padding)
    • Desktop: 1150-1919px (standard layout)
    • DesktopPlus: 1920px+ (expanded spacing)

Key Features

  • Background Image: Full-width with gradient overlay for text readability
  • Left Card: Features secondary (gold) button for primary offer visibility
  • Right Card: Uses primary (navy) button for scheduling/consultation CTAs
  • Responsive: Automatically adapts layout and typography for each breakpoint
  • Maximum Width: Content constrained to 1120px (Desktop) or 1400px (DesktopPlus)

Implementation Example

<section class="content-2column-graphic" 
         data-orientation="twocolumns" 
         data-breakpoint="desktop"
         style="background-image: url('path/to/your-image.jpg');">
    <div class="content-2column-graphic__inner">
        <div class="content-2column-graphic__grid">
            
            <!-- Primary Offer Card (Left) -->
            <div class="content-2column-graphic__card--left">
                <h2 class="card-heading">Your Headline</h2>
                <p class="card-body">Your compelling offer description.</p>
                <a href="/landing-page" class="card-button">CTA Text</a>
            </div>
            
            <!-- Secondary CTA Card (Right) -->
            <div class="content-2column-graphic__card--right">
                <h2 class="card-heading">Secondary Headline</h2>
                <p class="card-body">Brief call-to-action text.</p>
                <a href="/schedule" class="card-button">Book Call</a>
            </div>
            
        </div>
    </div>
</section>

CSS Variables Used

  • Primary colors: --primary-700, --primary-600, --primary-800
  • Accent colors: --accent-500, --accent-400, --accent-600
  • Neutral colors: --white, --neutrals-600
  • Typography: Lato (headings), Open Sans (body)

Integration Notes

  • Background images should be high-quality stock photos (1920x1080 minimum)
  • The gradient overlay ensures text remains readable regardless of image
  • Cards include subtle shadows for depth and visual hierarchy
  • Button styles match the atomic button component patterns
  • Component is fully responsive without JavaScript requirements
